# Funnelish Visual Interview Playbook

Use this playbook when you need to create beginner-friendly but advanced-looking Funnelish walkthroughs for interview prep, onboarding, or SOP documentation. Each task includes practical build steps, the expected result, common mistakes, evaluator checkpoints, and a visual capture brief for desktop screenshots, mobile screenshots, and short demo videos.

---

## Global Style Rules

Apply these defaults unless the funnel brief says otherwise:

- **Primary goal:** Keep the CTA visible, obvious, and repeated at natural decision points.
- **Desktop canvas:** 1440px wide preview.
- **Mobile canvas:** 390px wide preview.
- **Typography:**
  - Heading: 48px desktop / 30px mobile
  - Subheadline: 20px desktop / 16px mobile
  - Body: 16px
  - Button label: 18px semi-bold
- **Spacing:**
  - Section padding: 72px top and bottom on desktop
  - Section padding: 32px top and bottom on mobile
  - Element spacing: 12px to 20px vertical rhythm
- **CTA design:** High-contrast fill, rounded corners, minimum 48px tall, clear action label.
- **Proof mindset:** Whenever possible, pair the CTA with a product visual, a benefit, or urgency.

## 1) Create a Funnel Page from Scratch

### Objective
Build a clean landing page with a hero section, product image, and a prominent Buy Now button.

### Steps
1. In Funnelish, go to **Funnels → Create New Funnel**.
2. Select **Blank Funnel** so you can demonstrate true drag-and-drop skills.
3. Add a **Landing Page**.
4. Insert a new **Section** and then add a **Row** with a single column.
5. Drop in these elements from top to bottom:
   - Heading
   - Paragraph
   - Product image
   - Button
6. Use a conversion-first copy stack:
   - **Headline:** `Transform Your Morning Routine in 7 Days`
   - **Subheadline:** `A simple starter kit designed to make your skincare habit effortless.`
   - **CTA:** `Buy Now`
7. Center-align the content, cap the text width for readability, and give the button extra top margin.

### Expected Outcome
A clean above-the-fold hero where the headline communicates value fast, the product image anchors attention, and the CTA is impossible to miss.

### Common Mistakes
- Dropping elements directly into a section without understanding the section → row → element structure.
- Making the headline too long, which weakens the visual hierarchy.
- Letting the image or body text overpower the CTA.

### Pro Tips
- Keep the headline to one strong promise and the subheadline to one supporting sentence.
- If the layout looks empty, increase section padding before adding more elements.
- Add a second CTA further down the page later, not inside the hero immediately.

### Interviewer Checks
- Can the candidate create structure from scratch without a template?
- Do they understand how sections, rows, and elements relate?
- Is the CTA visually prioritized?

### Image Briefs and Video Storyboard (5–10s)
- **Desktop:** Full hero section with centered headline, product image, and CTA visible above the fold.
- **Mobile:** Stacked version with reduced font sizes and full-width CTA.
- **Storyboard:** Scroll from top of hero to CTA, then click **Buy Now**.

## 2) Full Width Section

### Objective
Turn a boxed layout into a true edge-to-edge section.

### Steps
1. Select the target **Section**.
2. Set **Section Width** to `100%`.
3. Enable **Stretch Section**.
4. Open the inner row settings and remove or increase any restrictive max-width.
5. Preview on desktop to confirm the background stretches fully.
6. Preview on mobile to make sure internal padding still keeps text readable.

### Expected Outcome
The background spans the full browser width while content stays aligned and intentional.

### Common Mistakes
- Enabling full width on the section but forgetting the inner row is still capped.
- Stretching everything edge-to-edge, including text, which hurts readability.
- Forgetting to recheck mobile padding after changing width behavior.

### Pro Tips
- Use full-width sections mostly for hero backgrounds, testimonial strips, or brand-colored CTA bands.
- Keep content in a controlled inner row even when the section background stretches.

### Interviewer Checks
- Does the candidate know where width constraints actually live?
- Can they diagnose a layout that still looks boxed after a width change?

### Image Briefs and Video Storyboard (5–10s)
- **Desktop before:** Section with visible 1200px-style boxed limitation.
- **Desktop after:** Full-width background with cleaner edge-to-edge presentation.
- **Storyboard:** Toggle stretch settings and preview the layout on desktop and mobile.

## 3) Mobile Responsive Design

### Objective
Adapt the funnel so it feels intentional on mobile instead of merely shrunken.

### Steps
1. Switch Funnelish to **Mobile View**.
2. Stack any two-column sections into a single-column flow.
3. Reduce headline and paragraph sizes for mobile readability.
4. Set product images to `100% width` or use a max width that fits the viewport.
5. Tighten section padding so key content appears sooner.
6. Hide decorative dividers, badges, or duplicate imagery if they push the CTA too far down.
7. Recheck button width and tap area.

### Expected Outcome
A mobile layout where the headline, product image, and CTA appear in a tight, thumb-friendly sequence.

### Common Mistakes
- Keeping desktop font sizes on mobile.
- Forgetting to collapse columns, leaving cramped side-by-side layouts.
- Hiding important conversion elements instead of purely decorative ones.

### Pro Tips
- Prioritize order: benefit, product, CTA, proof.
- On mobile, shorter content usually converts better than highly decorative layouts.

### Interviewer Checks
- Can the candidate think in mobile-first prioritization?
- Are they making responsive decisions that improve conversion, not just appearance?

### Image Briefs and Video Storyboard (5–10s)
- **Mobile:** Hero and product section stacked cleanly.
- **Desktop:** Matching desktop composition for comparison.
- **Storyboard:** Switch preview from desktop to mobile and show columns stacking.

## 4) Add & Customize Elements

### Objective
Add a heading, paragraph, button, and image, then style them with intention.

### Steps
1. Insert the four base elements into a row.
2. Update copy, image source, and button label.
3. Style the heading color, font family, and font weight.
4. Increase paragraph line-height and reduce opacity slightly for visual hierarchy.
5. Adjust button background color, text color, border radius, and padding.
6. Align all elements consistently left or center based on the section goal.
7. Tune spacing until the block feels balanced.

### Expected Outcome
The section feels branded and polished instead of looking like unstyled defaults.

### Common Mistakes
- Mixing alignment styles within the same content block.
- Using too many colors or fonts.
- Giving the image too much spacing above while crowding the CTA below.

### Pro Tips
- One accent color is usually enough.
- Make the button the brightest element in the block.

### Interviewer Checks
- Is the candidate comfortable with typography, spacing, and visual hierarchy?
- Can they improve a plain block without overdesigning it?

### Image Briefs and Video Storyboard (5–10s)
- **Before:** Default unstyled elements.
- **After:** Styled version with clearer hierarchy and CTA emphasis.
- **Storyboard:** Quick changes to button color, font size, and alignment, then hover over the CTA.

## 5) Duplicate & Reuse Section

### Objective
Duplicate an existing section and adapt it quickly.

### Steps
1. Right-click or open the action menu on a finished section.
2. Choose **Duplicate**.
3. Move the new section beneath the original.
4. Replace the copy, image, and CTA label to give it a new role.
5. Keep spacing and structure consistent so the page still feels unified.

### Expected Outcome
A second conversion block that looks consistent but delivers a different message, such as benefits, testimonials, or a repeated offer.

### Common Mistakes
- Forgetting to change duplicated placeholder copy.
- Making style changes that break visual consistency.
- Duplicating sections unnecessarily instead of reusing purposefully.

### Pro Tips
- Duplicate sections to accelerate testimonial cards, feature rows, guarantee blocks, or repeated CTA sections.
- Change only what needs to change: copy, image, and maybe background color.

### Interviewer Checks
- Does the candidate optimize for speed?
- Can they reuse structure while keeping the page coherent?

### Image Briefs and Video Storyboard (5–10s)
- **Comparison:** Original section beside or above the duplicated variant.
- **Storyboard:** Duplicate the section and swap in new content live.

## 6) Build Simple Product Section

### Objective
Create a product-focused block with image, name, price, urgency, and buy button.

### Steps
1. Add a new section under the hero.
2. Insert a two-column row.
3. Place the **Product Image** in the left column.
4. Place the following in the right column:
   - Product name
   - Short description
   - Price
   - Urgency text such as `Limited Offer`
   - Buy button
5. Use stronger font weight on the price and urgency label.
6. Add subtle shadow or border radius to the product image container if the style fits.
7. On mobile, stack the image above the purchase details.

### Expected Outcome
A product block that makes the offer easy to understand and easy to act on.

### Common Mistakes
- Burying the price below too much copy.
- Making urgency look spammy by overusing red labels or multiple countdown-like cues.
- Forgetting to stack image and text correctly on mobile.

### Pro Tips
- Put urgency near the CTA, not far above it.
- If space is tight, shorten the description instead of shrinking the button.

### Interviewer Checks
- Can the candidate build an offer section that feels purchase-ready?
- Do they know how to present product information in the right order?

### Image Briefs and Video Storyboard (5–10s)
- **Desktop:** Two-column product section.
- **Mobile:** Stacked product section.
- **Storyboard:** Hover over the product card or image, then click the buy button.

## 7) Add Custom HTML/CSS

### Objective
Use custom code to show comfort beyond the visual editor.

### Steps
1. Add a **Custom HTML** element.
2. Paste a simple example:

```html
<button style="background:red;color:white;padding:14px 28px;border:none;border-radius:999px;font-weight:700;cursor:pointer;">
  Buy Now
</button>
```

3. Preview the page.
4. If needed, wrap the button in a container with margin so it aligns with nearby content.
5. Explain when custom code is appropriate: custom styling, custom embeds, or advanced interactions not available in defaults.

### Expected Outcome
A rendered custom CTA that looks intentional and proves the builder can combine no-code and code-assisted workflows.

### Common Mistakes
- Forgetting padding, resulting in a tiny button.
- Using code that clashes with the page style.
- Adding custom code when the same result could be achieved faster with native controls.

### Pro Tips
- Use custom HTML sparingly in time-boxed interview tasks.
- Mention that maintainability matters; native Funnelish controls are usually easier to edit later.

### Interviewer Checks
- Does the candidate understand when custom code is useful versus unnecessary?
- Can they keep the custom output visually consistent with the rest of the page?

### Image Briefs and Video Storyboard (5–10s)
- **Image:** Custom-coded button rendered inside the layout.
- **Storyboard:** Paste the code, preview it, and click the button.

## 8) Fix Layout Problem

### Objective
Diagnose and fix spacing and alignment issues quickly.

### Steps
1. Identify the symptom first: too much empty space, uneven alignment, or cramped content.
2. Inspect the structure in this order:
   - Section
   - Row
   - Column
   - Element
3. Reduce or add margin and padding based on where the issue actually lives.
4. Recheck alignment settings in the row and individual elements.
5. Preview in both desktop and mobile before calling the fix done.

### Expected Outcome
A cleaner layout with tighter spacing, aligned elements, and fewer distractions.

### Common Mistakes
- Changing multiple levels of spacing at once and losing track of the real cause.
- Fixing desktop spacing but breaking mobile spacing.
- Confusing margin with padding and overcorrecting.

### Pro Tips
- When in doubt, change one spacing control at a time.
- Large empty areas often live on the section, not the element itself.

### Interviewer Checks
- Can the candidate troubleshoot methodically?
- Do they understand which container controls what?

### Image Briefs and Video Storyboard (5–10s)
- **Before:** Misaligned or overly spaced section.
- **After:** Corrected spacing and alignment.
- **Storyboard:** Adjust margin/padding live and show the result immediately.

## 9) Multi-Step Funnel

### Objective
Build a simple three-step funnel: landing, checkout, thank-you.

### Steps
1. Create three pages inside the same funnel:
   - **Page 1:** Landing
   - **Page 2:** Checkout
   - **Page 3:** Thank You
2. On the landing page, build a focused hero with CTA.
3. Link the CTA to the checkout page.
4. On the checkout page, keep distractions low and show order details clearly.
5. On the thank-you page, confirm the order and offer the next action, such as a bonus, onboarding step, or upsell.
6. Test the page flow end to end.

### Expected Outcome
A working three-step journey where the visitor moves cleanly from offer to purchase to confirmation.

### Common Mistakes
- Linking the CTA incorrectly.
- Leaving main-site navigation or extra exit points on the checkout step.
- Making the thank-you page feel like a dead end.

### Pro Tips
- Give each page one job.
- Use consistent branding across all three steps.

### Interviewer Checks
- Does the candidate understand funnel progression instead of isolated page design?
- Can they connect steps logically and test navigation?

### Image Briefs and Video Storyboard (5–10s)
- **Desktop images:** Landing, checkout, and thank-you pages.
- **Mobile images:** Matching three-step mobile previews.
- **Storyboard:** Click from landing to checkout to thank-you.

## 10) Speed Task

### Objective
Demonstrate the ability to build a simple, conversion-ready page in 10–15 minutes.

### Suggested Build Order
1. Hero section
2. Product section
3. Repeated CTA block
4. Mobile adjustments
5. Final spacing cleanup

### Expected Outcome
A minimal but credible funnel page that looks intentional, responsive, and purchase-focused.

### Common Mistakes
- Spending too long on decoration before the core structure exists.
- Perfecting one section while other key sections are missing.
- Forgetting to test mobile before time runs out.

### Pro Tips
- Reuse sections aggressively.
- Use one accent color, one product image style, and one CTA style.
- Narrate your decisions if the task is live: speed with clarity looks strong.

### Interviewer Checks
- Can the candidate prioritize high-impact work under time pressure?
- Do they know what to skip?

### Image Briefs and Video Storyboard (5–10s)
- **Video demo (time-lapse concept):** Show the drag-and-drop workflow from blank page to finished simple funnel in a compressed 10–15 second montage.

## Recommended Deliverable Format for AI-Generated Outputs

When asking an AI assistant to produce these walkthroughs, require the following format for every task:

1. **Task Name**
2. **Goal**
3. **Step-by-Step Instructions**
4. **Expected Outcome**
5. **Common Mistakes**
6. **Pro Tips**
7. **Interviewer Checks**
8. **Desktop Image Brief**
9. **Mobile Image Brief**
10. **5–10s Video Storyboard**

This structure keeps the guidance beginner-friendly while still showing advanced thinking around conversion, hierarchy, and responsive behavior.
